Story Mode README

目的

このディレクトリは、GUNDAM BATTLE SLOT のストーリーモード専用仕様を管理する。

ストーリーモードは通常対戦とは別に、以下の専用要素を持つ。

- クリエイトガンダム系列の育成
- クリエイトガンダムラボ
- スロット入れ替え
- 装備品管理
- エネルギー管理
- リロード武器管理
- 同行機体管理
- 同行機体育成
- ストーリー進行フラグ
- チャプター別戦闘定義
- 報酬管理

---

推測禁止

ストーリーモード実装時も、既存開発ルールに従う。

- 実コード未確認で既存処理を前提にしない
- 未確定の数値を仮置きしない
- 説明文依存で内部処理を作らない
- 仕様変更時はこの README または同階層の仕様ファイルを更新する
- 実装時は最新の "index.html" / "js" 実コードを確認する

初期特殊行動

特殊行動1. リロード

行動権を消費して、リロードが必要な武器を指定してリロードする。

リロードは一括ではなく、武器種指定で行う。

特殊行動2. エネルギーチャージ

行動権を消費して、エネルギーの充填を行う。

特殊行動3. 装備品1

装備品を使用する。

特殊行動4. 装備品2

装備品を使用する。

装備品は最大2個まで。

特殊行動5. エネルギー調整

エネルギー武器のエネルギーを同量追加消費して、ダメージ値を元の値の1.5倍へ増加させる。

加算累積可能。

クリエイトガンダム系列共通仕様

各スロットは、条件を満たすとクリエイトガンダムラボで入れ替え可能。

クリエイトガンダム系列は以下を持つ。

- HP
- 回避ストック
- エネルギー
- 使用可能コスト
- スロット構成
- 装備品
- クリエイトスキル

HP、回避ストック、エネルギー、武器性能、装備品、スロット登録は、使用可能コストの範囲内で構築する。

---

リロード武器仕様

リロード武器は、ダメージ値・攻撃回数とは別に弾数を管理する。

例：

汎用マシンガン
10ダメージ × 3回
射撃
リロード
弾数10
1ターン1装填

この場合、1回選択すると

10ダメージ × 3回を発動
弾数10 → 7

となる。

リロード属性表示

リロード属性は "[R]" で表記する。

ターン経過リロード

武器種ごとにターン経過時の装填値を持つ。

例：

1ターン1装填
2ターン1装填
5ターン1装填

特殊行動リロード

特殊行動「リロード」は、リロード対象の武器種を指定して行う。

---

リロード武器の追撃仕様

弾数が余っている場合、所持回避1消費で追撃可能。

回避が0の状態でも、1回だけ追撃可能。

回避0で追撃した場合、その追撃の会心率は倍になる。

---

エネルギー武器仕様

クリエイトガンダム系列はエネルギーを持つ。

プロトクリエイトガンダムの初期エネルギーは100。

エネルギー初期値は条件で増減する。

エネルギー回復

プロトクリエイトガンダムは、1ターンにエネルギーを5回復する。

エネルギー武器

エネルギー武器は、攻撃時に設定されたENを消費する。

エネルギーが0の場合は不発となる。

エネルギー調整

特殊行動「エネルギー調整」により、エネルギー武器の消費ENを増やしてダメージを増加できる。

この調整は事前設定型とする。

実際のスロット選択時には、調整後の数値を参照する。

---

同行機体仕様

同行機体もストーリーモード内で育成可能。

各同行機体は以下を持つ。

- 解放状態
- 同行可能状態
- レベル
- Exp
- ストーリー専用補正
- 装備または強化状態

同行機体は通常対戦とは別管理とする。

---

ストーリー限定レベル補正

ストーリーモードでは各機体ごとにレベルを持つ。

レベル上昇時の恩恵：

レベル値の半分の数値が会心率に蓄積
レベル値の半分の数値分、ダメージを軽減

例：

Lv2  → 会心率 +1 / ダメージ軽減1%
Lv10 → 会心率 +5 / ダメージ軽減5%
Lv50 → 会心率 +25 / ダメージ軽減25%

ストーリーモード実装土台設計書

1. 目的

ストーリーモードは、プレイヤーがクリエイトガンダム系機体を育成・改修しながら、各チャプターの戦闘を突破していく専用モードである。

通常対戦とは別に、以下の専用要素を持つ。

- ストーリー進行フラグ
- クリエイトガンダムラボ
- 機体レベル
- Exp
- 使用可能コスト
- スロット登録
- 装備品管理
- 同行機体管理
- 同行機体育成
- ストーリー専用戦闘
- チュートリアル・AI会話イベント

---

2. 最初に作るべき土台

2-1. ストーリー進行データ

管理対象：

- 現在チャプター
- クリア済みイベント
- 解放済み機能
- 設計図入手状況
- AI会話分岐
- プレイヤー目標
- 重要選択肢

想定データ：

storyState = {
  chapter: 1,
  clearedEvents: [],
  flags: {},
  unlockedFeatures: {},
  blueprints: {},
  goalText: ""
};

---

2-2. ストーリー機体データ

クリエイト系と同行機体を管理する。

storyUnits = {
  createUnits: {},
  companionUnits: {}
};

各機体が持つ情報：

unitStoryData = {
  unitId: "",
  level: 0,
  exp: 0,
  unlocked: false,
  companionAvailable: false,
  storyOnly: true,

  maxCost: 0,
  usedCost: 0,

  hpCost: 0,
  evadeCost: 0,
  weaponCost: 0,
  equipmentCost: 0,

  registeredSlots: [],
  equippedItems: []
};

重要方針：

- クリエイトガンダム系だけでなく、同行機体も育成可能。
- 機体レベル上昇で使用可能コストが増える。
- HP、回避ストック、武器性能、装備品、スロット登録はコスト範囲内で管理する。

---

3. レベル・Exp・コスト設計

3-1. レベルシステム

ストーリーモード限定で、各機体ごとにレベルを持つ。

レベル上昇効果：

- 使用可能コスト増加
- 会心率補正
- ダメージ軽減補正

既定方針：

- レベル値の半分を会心率に蓄積
- レベル値の半分の数値分、ダメージ軽減

例：

Lv2  → 会心率 +1相当 / 軽減1%
Lv10 → 会心率 +5相当 / 軽減5%
Lv50 → 会心率 +25相当 / 軽減25%

※ 実際の数値式は後で確定。

---

3-2. コストシステム

レベルに応じて使用可能コストが増える。

storyLevelTable = {
  0: { expToNext: 0, maxCost: 0 },
  1: { expToNext: 0, maxCost: 0 }
};

コスト対象：

- HP強化
- 回避ストック強化
- 武器性能強化
- スロット登録
- 装備品
- 特殊行動強化

方針：

強さは、機体レベルで増えたコスト枠の中で組む。

---

4. クリエイトガンダムラボ

ラボで行うこと：

- クリエイトガンダム各種選択
- 同行機体選択
- 装備品選択
- コスト振り分け
- スロット登録
- 強化内容確認
- 保存
- リセット

対象機体：

- プロトクリエイトガンダム
- クリエイトガンダム
- クリエイトⅠガンダム
- クリエイトXガンダム
- クリエイトXXガンダム
- クリエイトX²ガンダム

同行可能数：

プロトクリエイトガンダム：2機
クリエイトガンダム：4機
クリエイトⅠガンダム：6機
クリエイトXガンダム：10機
クリエイトXXガンダム：20機
クリエイトX²ガンダム戦：選択可能同行機すべて

---

5. 報酬テーブル

storyRewards = {
  enemyId: {
    exp: 0,

    unlockSlots: [],

    unlockFlags: []
  }

報酬対象：

- Exp
- 同行機体解放
- スロット行動
- 装備品
- 設計図
- チャプター進行フラグ

---

6. ストーリー戦闘定義

チャプターごとの戦闘を定義する。

storyBattles = {
  battleId: {
    chapter: 1,
    title: "",
    mode: "",
    playerUnits: [],
    enemyUnits: [],
    enemyLevel: 0,
    allowCompanion: false,
    forcedCompanion: [],
    clearEvent: "",

  }

必要な戦闘形式：

- 1on1
- 2on2
- 2vs1
- ボス戦
- チュートリアル戦
- ミラーマッチ
- 同行機体解放戦

10. 重要方針

- 最初から全チャプターを実装しない。
- まずはチャプター1、ラボ解放、育成データ保存までを完成させる。
- 数値は未確定なら仮置きしない。
- 実コード追加時は、最新HTML/JSを確認してから行う。
- ストーリー専用データは通常対戦データと混ぜない。
- クリエイトガンダム系、同行機体、報酬、戦闘定義は別ファイル管理にする。この資料を土台にするなら、次に決めるべきは 追加ファイル構成 です。
たとえば js_story_state.js、js_story_units.js、js_story_lab.js、js_story_battles.js みたいに分けるのが安全です。
